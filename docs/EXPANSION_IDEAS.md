## 2. AR 拡張アイデア

### コンセプト

AR ＝ **見えない感覚を「見せる」出力。** デバイスが感じた距離・熱・方角・AI認識をカメラ映像に重畳する。装着者だけでなく**観客も第六感を覗ける → デモ映えに直結**。「狙った先に意思を届ける」という体験とAR照準が噛み合う。

### 前提（重要）

SPRESENSE自体は描画できない → **スマホ／ブラウザを表示装置にする。**
Reactスタックがあるので **WebAR** が自然で、新フレームワーク不要。

```
SPRESENSE（感覚） → サーバー → WebARページ（スマホのカメラ映像に重畳）
```

### Tier 別アイデア（実現性順）

**Tier 1 — HUD型オーバーレイ（最も現実的・Reactで完結）**
- センサー値をカメラ映像に重ねる：距離・方角・温度・ポーズを画面表示
- 超音波の検知コーンを可視化 → 観客が「今これを察知している」と分かる
- 仕組み：WebRTCでカメラ取得 → canvasで重ね描き。**世界固定なしのHUD**なので難易度低

**Tier 2 — マーカーAR（AR.js・無料・ブラウザ）**
- ラジコンや対象にマーカーを貼る → 3Dグラフィックやロックオン枠が"世界に刺さる"
- 「狙う」テーマを視覚化

**Tier 3 — AI × AR**
- かざす → 撮影 → Gemini認識 → **ARラベルが対象にピン留め**

**Tier 4 — 操作AR（保険にもなる）**
- ジェスチャーでAR物体を操作。物理ラジコンの**仮想版**＝かざしてARカー／ドローンを操縦
- 物理ラジコンが間に合わない場合の**代替デモ**にもなる

**現状のおすすめ:** Tier 1 のHUDオーバーレイから。
カメラ＋canvasに重ねるだけでReact完結・新ツール不要。既に取れている距離・ポーズを流すだけで成立。世界固定が欲しくなったら Tier 2（AR.js）へ拡張。
